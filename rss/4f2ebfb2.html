瑞幸 Skill 引发的一些思考

瑞幸也出 skill 了，引发了我对 AI 时代和互联网时代的一些思考。

skill 叫 My Coffee Skill，一个 zip 包，装进 Claude Code、Codex、Cursor，你跟 agent 说句"来杯热美式，到公司自提"，它就帮你下单。

服务正在和平台脱钩

过去你点咖啡绕不开平台，美团也好，某个超级 App 也好，它都是那个"你去哪点"的中心。平台值钱，值在它聚合了供给、攥着流量入口。

可当瑞幸、肯德基、蜜雪自己把服务做成 skill，每个 agent 都能直连，这个中心就被绕过去了。你不再"进一个 App 点单"，你是"让你的 agent 去点"。服务和平台之间那根绳，松了。

而且瑞幸根本不用挑边。它写的是一个 skill，一份 SKILL.md 加几个脚本，本身和 agent 无关。同一个包，Claude Code、Codex、Cursor、Qoder、Kimi 全都能读，多支持一个 agent 几乎不花额外成本。

这才是关键：服务之所以能这么轻地脱开平台，是因为 skill 天生就是"写一次、到处能用"的。过去要给每个平台单独做一套对接，又贵又被绑；现在一份能力，所有 agent 通吃。平台不再是必经之路，因为绕开它几乎没有成本。

平台为什么不能垄断入口？

你可能会想：服务脱了平台，那 agent 自己不就成了新的垄断入口吗？谁先把所有人圈进自己的 agent，谁不就是下一个巨头？

这是互联网时代的惯性思维。它过不去一道坎——token。

回头看互联网那二十年，抢入口、抢流量是铁律。道理很简单：多服务一个用户几乎不花钱。零边际成本，用户越多，成本摊得越薄，规模本身就是护城河。所以所有人挤破头抢日活、抢入口，做大就是赢。

Agent 时代，这条铁律反了过来。每一次对话都在烧 token，而 token 就是钱。你的流量涨一倍，烧的钱也涨一倍。流量不再是能无限摊薄的资产，它本身就是成本。

这一下，"做大入口"从印钞变成了烧钱。你圈进来的人越多，账亏得越狠。所以"垄断入口"这件事，在 agent 时代根本立不住：不是没人想垄断，是做大本身就会把自己拖垮。

互联网抢入口的时代，不是被哪家公司终结的，是被 token 这条底层逻辑直接判了结束。

所以入口不会再中心化成一个超级 App。它会碎，会往下沉——沉到每个人自己那一个 agent 上。

App 不会进化，但 agent 会

到这儿有人会抬杠：那跟现在用 App 有啥本质区别？不还是登录、支付、自提，一步没少？

流程确实一样。但盯着流程看，就看丢了重点。重点不是界面从"点按"变成了"对话"——是 App 不会按你进化，agent 会。

你手机里那个瑞幸 App，和别人手机里的是同一个；你用一年，它还是那个样子，不认识你，也不会因为你而变。agent 反过来：你越用，它越合你的身，记得你的口味，到点主动问你要不要来一杯，还能把咖啡、日历、打车几个 skill 串起来，一句"明早八点航班"就替你拆成订车、点咖啡、机场自提。它不是一件用完放下的工具，是一个会照着你不断改形状的东西。

说白了：App 是做给"所有人"的，一经发布就定型；agent 是做给"你一个人"的，而且一直在按你改。这才是分水岭。

可它不会凭空就懂你。这背后是一道更慢的功夫——信任，和时间。

和你的 agent 慢慢建立信任

agent 再聪明，也需要有和你建立信任的过程。要让它不用你每次点确认就替你花钱、用你的身份跨几十个品牌下单，你得把钱包交给它。这不是技术问题，是信任问题。而信任，是养出来的，不是一次给的。

没人第一天就交钥匙。你先让它点杯咖啡这种小事，对了几十次，你才敢托付更大的。中间你顺手把"我只喝热的、到公司自提"沉淀成一条规则，它的权限边界就一点点放开。和带一个新助理，没有任何区别。

用得越久，你和它之间攒下的规矩越多，它对你就越聪明。不是模型变强了，是你们一起攒下的那本"你的说明书"厚了。

而你辛辛苦苦攒下的这套规矩、这份记忆，是绑死在某一个 agent 上的。换一家，从头再来。这才绕回开头那个平台问题：新的护城河，不再是平台聚合了多少供给，而是你的规矩住在谁家。谁存着你的说明书，谁就锁住了你。平台没消失，它只是从"大家共用的入口"，变成了"你私人的那一个"。

一杯咖啡变成一个 skill。瑞幸大概只想多卖几杯，但它顺手让我看清了未来几年的走向：服务脱钩平台、挂上 agent；token 这条底层逻辑堵死了新的垄断；终点不是一个更强大的超级 App，是一个你亲手养大、足够信任、替你连接整个世界的 agent。

贾维斯不是被造出来的。是被你，一天一天，喂出来的。
