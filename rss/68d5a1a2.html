Spring Boot 3.5.3 现已发布

Spring Boot 3.5.3 现已发布，现在可以从 Maven Central 获取。此计划外版本修复了 Spring Boot 3.5.1 中意外引入的一个严重回归问题，该问题在 Spring Boot 3.5.2 中尚未完全修复。

v3.5.3 错误修复

  • Binder 上下文无法恢复先前的源，导致 Spring Boot 3.5 或更高版本上的数据丢失 #46040

v3.5.1

Noteworthy Changes

  • 此版本升级至 Tomcat 10.1.42，其中引入了对multipart/form-data 请求中 part count h和 header size 的限制。这些限制可分别使用server.tomcat.max-part-count和server.tomcat.max-part-header-size进行自定义。

New Features

  • 允许在 ConfigDataEnvironmentContributors 上指定 ConfigData.Options #42932

Bug Fixes

  • 当 classpath URL 引用 host 时，可执行 JAR 应用程序类会遇到性能问题#46028
  • 当 TCCL 在调用之间发生变化时，从 spring.factories 加载可能会失败，并出现 ClassNotFoundException 异常#46019
  • spring.couchbase.authentication.jks.private-key-password 无效#46006
  • Actuator heapdump endpoint 在现代 OpenJ9 JVM 上失效#46005
  • IndexedElementsBinder 不再抛出 UnboundConfigurationPropertiesException #45994
  • DataSouceBuilder 在驱动程序为空时可能导致 NPE 失败#45992
  • JSON writer 错误地转义了正斜杠，这可能会导致结构日志问题#45980
  • ManagementContextAutoConfiguration 添加了一个降低绑定性能的属性源#45968
  • 当“java.net.http.HttpClient”不可用时，ClientHttpConnectorAutoConfiguration 加载失败#45955
  • 无法选择退出配置文件验证或使用包含“.”的配置文件名称#45947
  • GraphQlProperties.DeprecatedSse 未被注释为已弃用#45878
  • 读取 MANAGEMENT_SERVER_PORT 时，SpringApplication.setEnvironmentPrefix 被忽略#45857
  • 由于 CSRF 保护，写入和删除操作不再适用于 Spring Security 支持的 Cloud Foundry 执行器#45848
  • Binding 不再适用于非大写的系统环境属性#45741
  • ManagementWebServerFactoryCustomizer 和 ManagementErrorPageCustomizer 不应该具有相同的顺序#45736
  • Awailitility 的默认版本与 Kotlin 1.9 baseline 不兼容#45673
  • Spring Boot 3.5 的依赖管理应该升级到 Lettuce 6.6.0.RELEASE #45670
  • Spring Boot 3.5 的依赖管理应该升级到 Jedis 6.0.0 #4566
  • Spring Boot 3.5 的依赖管理应该升级到 MongoDB 5.5.0 #45660
  • ……

更多详情可参阅 3.5.1 博客公告和 3.5.1 发行说明。
